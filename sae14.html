Se présenter sur Internet

Problème 1 — Une image ne s’affichait pas sur le site web

Introduction

Dans le cadre de la SAÉ14, j’ai développé un site web présentant plusieurs aspects culturels et touristiques de la Tunisie. Le projet comportait plusieurs pages HTML ainsi qu’un grand nombre d’images utilisées pour illustrer les différentes destinations et spécialités tunisiennes.

Lors des tests du site web, j’ai rencontré un problème d’affichage concernant certaines images du projet.

Apparition du problème

Après avoir intégré une image dans une page HTML, celle-ci ne s’affichait pas correctement dans le navigateur.

À la place de l’image, le navigateur affichait une image cassée ou un espace vide.

Figure 1 — Image absente sur la page web.

Analyse du problème

Après analyse du code HTML, j’ai constaté qu’une erreur était présente dans le chemin de l’image utilisé dans la balise :

<img src="image_projet/couscouus.jpg">

Le nom du fichier indiqué dans le code ne correspondait pas exactement au fichier présent dans le dossier du projet.

Le problème provenait donc d’une erreur de chemin d’accès et non du navigateur ou du serveur web.

Résolution du problème

Pour corriger cette erreur, j’ai vérifié le contenu du dossier image_projet puis corrigé le nom du fichier dans le code HTML :

<img src="image_projet/couscous.jpg">

Après cette correction, l’image s’est affichée correctement sur la page web.

Figure 2 — Affichage correct de l’image après correction du chemin.

Problème 2 — Le fichier CSS ne se chargeait plus correctement

Introduction

Afin d’améliorer l’apparence et l’ergonomie du site web, une feuille de style CSS commune a été utilisée pour personnaliser l’ensemble des pages du projet.

Lors des tests du site web, j’ai rencontré un problème empêchant le chargement du style CSS sur une des pages du projet.

Apparition du problème

En ouvrant une page du site web dans le navigateur, les éléments HTML apparaissaient sans mise en forme.

Les couleurs, les polices, les espacements et la mise en page n’étaient plus appliqués correctement.

Figure 3 — Page web affichée sans style CSS.

Analyse du problème

Après analyse du code HTML, j’ai constaté qu’une erreur était présente dans le lien vers la feuille de style CSS.

Le fichier était appelé avec un mauvais nom dans la balise :

<link rel="stylesheet" href="styles.css">

Alors que le fichier du projet s’appelait :

style.css

Le navigateur ne parvenait donc pas à charger la feuille de style du projet.

Résolution du problème

Pour corriger cette erreur, j’ai modifié le lien dans le fichier HTML afin d’utiliser le bon nom de fichier :

<link rel="stylesheet" href="style.css">

Après cette correction, l’ensemble des styles CSS a été correctement appliqué à la page web.

Figure 4 — Affichage correct de la page après chargement du CSS.
